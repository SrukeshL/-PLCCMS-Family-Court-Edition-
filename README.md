## 🏛️ What Is This?

The **Parent/Lawyer Case Custody Management System (PLCCMS)** is a free, open-source, privacy-first application for organizing and managing family court custody cases. It converts a structured Excel workbook into a professional, court-ready HTML application — running entirely on your own computer, with no data ever sent to any server.

### The Problem It Solves

> A typical family court judge handles **200+ cases per week.**
> At 8 hours/day × 5 days = 2,400 minutes ÷ 200 cases = **12 minutes per case.**
> That is 12 minutes to review evidence, hear both parties, and decide a child's future.

This application is specifically designed to make those 12 minutes count — ensuring every fact, every piece of evidence, and every legal argument is organized, prioritized, and court-ready **before** the hearing begins.

## 🚀 Quick Start

### Prerequisites
- Python 3.8 or higher — [python.org/downloads](https://python.org/downloads) — check ✅ **"Add Python to PATH"**
- Microsoft Excel or [LibreOffice Calc](https://www.libreoffice.org) (free)
- Any modern web browser

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/Srukesh L/PLCCMS.git
cd PLCCMS

# 2. Install required Python libraries
pip install -r requirements.txt

# 3. Open Excel template — fill in yellow cells only
#    family_court_custody_template.xlsx

# 4. Generate your application
python generate_dashboard.py

# 5. Open in browser
# Windows:  start family_court_dashboard.html
# Mac:      open family_court_dashboard.html
# Linux:    xdg-open family_court_dashboard.html
```

## 🔄 Single vs. Dual Party Mode

```
SINGLE PARTY (default)                     DUAL PARTY (when both participate)
──────────────────────                     ──────────────────────────────────
Your Excel → Your App                      Your Excel + Their Excel
Party B columns ready & waiting        →   merge_dashboards.py
Court note auto-generated              →   Judge sees unified comparison
Other party can join ANY time          →   Agreements green / Disputes red
No data ever lost switching modes      →   All existing data preserved
```

### Switch from Single to Dual — 3 Steps, Zero Data Loss

```
1. Share family_court_custody_template.xlsx with the other party
2. In Dashboard Config: Party Mode = DUAL | Party B Excel File = their_file.xlsx
3. Run: python generate_dashboard.py   ✅ Done
```

## ⚖️ Pennsylvania Custody Factors — 23 Pa.C.S. § 5328

**Amended by Act 11, June 30, 2025 — Effective August 2025**

### 🔴 Safety Factors — Substantial Weighted Consideration

| Factor | Legal Standard |
|--------|---------------|
| **(1)** | Which party more likely to ensure the **safety of the child** |
| **(2)** | Present and past **abuse** — including PFA and SVPO orders with abuse findings |
| **(2.1)** | **Child abuse and protective services** involvement — § 5329.1(a) |
| **(2.2)** | **Violent or assaultive behavior** by a party |

### All Other Factors

| Factor | Legal Standard |
|--------|---------------|
| **(2.3)** | **Cooperation and conflict** — encouraging contact; parental alienation |
| **(3)** | **Willingness and ability** — care, stability, continuity, daily needs |
| **(4)** | **Stability and continuity** in education, family, community life |
| **(6)** | **Sibling and familial** relationships |
| **(7)** | **Child's well-reasoned preference** — maturity and judgment |
| **(11)** | **Proximity** of residences |
| **(12)** | **Employment schedule** and childcare arrangements |
| **(14)** | History of **drug or alcohol abuse** |
| **(15)** | **Mental and physical condition** of party or household member |
| **(16)** | **Any other relevant factor** |

### Special Provisions
| Provision | Description |
|-----------|-------------|
| **§ 5328(a.1)** | Abuse Exception — circumstances in response to abuse NOT weighed against protecting party |
| **§ 5328(a.2)** | Totality Rule — no single factor is determinative |
| **§ 5328(b)** | Gender Neutral — no preference based on gender |
| **§ 5328(d)** | 30-Day Notice — court must provide § 5328 to all parties within 30 days of filing |

## 📊 Priority & Confidentiality System

### Five-Level Priority System

| Level | When to Use |
|-------|-------------|
| 🟢 **LOW** | Background or supporting context |
| 🔵 **MEDIUM** | Notable and relevant to the case |
| 🟠 **HIGH** | Significant impact on custody determination |
| 🔴 **CRITICAL** | Directly affects the custody decision |
| ⚫ **CATASTROPHIC** | Immediate child safety — urgent court action required |

## ⚖️ ODC Professional Conduct Module

### Why This Module Exists — Building Trust, Not Suspicion

The Professional Conduct Reference was included primarily to **strengthen** the attorney-client relationship. Here is the full reasoning:

**1. Trust Is the Foundation**
Custody cases require parents to share their most sensitive personal information with their attorney. Parents share fully only when they trust completely. Showing parents what rules their attorney has sworn to follow says: *"Here is what your lawyer promised you."* This builds confidence.

**2. Transparency Protects Both Sides**
When parent and attorney see the same rules in the same application, misunderstandings about fees, communication frequency, and scope are resolved before they become disputes — protecting both the client and the attorney.

**3. Prevents Problems Before They Start**
Most ODC complaints arise from miscommunication, not genuine misconduct. Showing Rule 1.4 (Communication), Rule 1.5 (Fees), and Rule 1.16 (Withdrawal) upfront sets correct expectations on both sides before problems arise.

**4. Protects Good Attorneys Too**
The structured tracker requires specific dates, specific rules, and specific evidence meeting the *clear and convincing evidence* standard. This protects honest attorneys from vague or emotional complaints while ensuring genuine violations are properly documented.

**5. Levels the Knowledge Imbalance**
Parents in custody cases are often emotionally overwhelmed and legally inexperienced. Having professional conduct rules visible in the same application they use daily informs the relationship — not adversarially, but transparently.

**6. Benefits the Entire Proceeding**
Attorney misconduct affecting court proceedings can be documented by any party — the client, opposing counsel, or the judge — raising the standard of practice across the entire case.

**7. Aligns With Court Modernization**
The PA Supreme Court and ODC's own materials encourage transparency and client education. Including professional conduct rules aligns this application with values the court system already endorses.

**8. Solves a Real Documentation Problem**
ODC inquiry letters show most complaints are poorly documented — missing timelines, no evidence, no fee agreement copies. This tracker solves the documentation problem while the attorney-client relationship is still active.

> *"The Professional Conduct Reference is included to strengthen the attorney-client relationship —*
> *not to undermine it. When both parent and attorney understand the same professional obligations,*
> *communication improves, expectations align, and focus stays where it belongs:*
> *on the best outcome for the child."*

### Supported AI Tools

| AI Tool | Provider | Subscription Recommendation |
|---------|----------|-----------------------------|
| **Claude** | Anthropic | Claude Pro for large case files |
| **ChatGPT** | OpenAI | ChatGPT Plus for large files |
| **Google Gemini** | Google | Google One AI Premium |
| **Microsoft Copilot** | Microsoft | Microsoft 365 Copilot |

> 💡 **Subscription Note:** For large case volumes — hundreds of emails, lengthy transcripts,
> multiple court filings — a paid AI subscription ($20–$30/month) is strongly recommended.
> Free tiers have context limits that may produce incomplete results.
> The cost is a fraction of one hour of attorney time. Cancel when your case concludes.

### What AI Can Help With

| Task | How AI Assists |
|------|---------------|
| **Validate evidence entries** | Check if evidence supports the custody factor cited |
| **Review cross-exam questions** | Suggest follow-up questions and identify gaps |
| **Summarize communications** | Condense email threads into clear timeline entries |
| **Plain to court language** | Translate everyday language into court-appropriate formal language for filings and submissions |
| **Draft ODC narrative** | Structure factual account clearly and chronologically |
| **Check factor coverage** | Identify which § 5328 factors have weak or no evidence |
| **Review judge's summary** | Ensure it is factual, clear, and concise |
| **Negotiation positions** | Identify potential compromise zones based on factor analysis |

### What AI Must NOT Be Used For

| Task | Reason |
|------|--------|
| Predicting case outcome | AI cannot know your judge, local practices, or full context |
| Legal strategy decisions | Requires a licensed attorney |
| Confirming case facts | AI can hallucinate — always independently verify |
| Medical, financial, or sealed documents | Never paste these into any AI tool |
| Permanent case storage | Delete chat after every session — no exceptions |

### Privacy Protocol — Follow Every Session

```
✅ Use temporary/incognito chat mode when available
✅ Use child's initials only — never full name
❌ Never paste medical, financial, or sealed records
❌ Never paste attorney-client privileged content
✅ Delete conversation immediately after session ends
```

| Tool | How to Delete |
|------|--------------|
| **Claude** | Settings → Delete Conversation or use temporary chat |
| **ChatGPT** | Sidebar → three dots → Delete |
| **Gemini** | myactivity.google.com → Delete activity |
| **Copilot** | Three dots → Clear conversation |
